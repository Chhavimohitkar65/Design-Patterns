Proxy Design Pattern

Definition

The Proxy pattern provides a surrogate or placeholder for another object to control access to it. It creates a representative object that controls access to another object, which may be remote, expensive to create, or in need of securing.

User Controls

System Activity

How This Example Works

This demonstration shows a secure file system using the Proxy pattern:

  • Real Subject (FileSystem): The actual file system that performs file operations
  • Proxy (SecureFileSystem): Controls access to the file system, checking user authentication
  • Client: The user interface that interacts with the file system through the proxy
// Real Subject class FileSystem { accessFile(filename) { return `Reading file: ${filename}`; } } // Proxy class SecureFileSystem { constructor() { this.fileSystem = new FileSystem(); this.isAuthenticated = false; } authenticate() { this.isAuthenticated = true; } logout() { this.isAuthenticated = false; } accessFile(filename) { if (!this.isAuthenticated) { throw new Error("Access denied. Please login first."); } return this.fileSystem.accessFile(filename); } }
