Iterator Design Pattern

A behavioral design pattern that provides a way to access elements of a collection sequentially without exposing its underlying representation.

Definition

The Iterator pattern allows you to traverse elements of a collection without exposing its underlying representation (list, stack, tree, etc.). It extracts the traversal behavior of a collection into a separate object called an iterator.

// Iterator Interface
class Iterator {
    hasNext() {}
    next() {}
    current() {}
    hasPrevious() {}
    previous() {}
}

// Concrete Iterator Example
class SequentialIterator extends Iterator {
    constructor(collection) {
        super();
        this.collection = collection;
        this.position = 0;
    }

    hasNext() {
        return this.position < this.collection.length;
    }

    next() {
        return this.hasNext() ? 
            this.collection[this.position++] : null;
    }
}
