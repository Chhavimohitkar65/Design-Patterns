Adapter Pattern Explanation

The Adapter pattern converts the interface of one class into another interface that clients expect. It lets classes work together that couldn't otherwise because of incompatible interfaces.

// Target Interface (EU Socket)
interface EUSocket {
    void plugIn();
}

// Adaptee (US Device)
class USDevice {
    void powerOn() {
        // US specific power implementation
    }
}

// Adapter
class UStoEUAdapter implements EUSocket {
    private USDevice device;

    public UStoEUAdapter(USDevice device) {
        this.device = device;
    }

    public void plugIn() {
        // Convert EU socket standard to US
        device.powerOn();
    }
}

In this example:

  • The US Device (Adaptee) has an incompatible interface with the EU Socket (Target)
  • The Adapter converts the US plug to work with the EU socket
  • The client can use the US Device with the EU Socket without knowing about the adaptation process
