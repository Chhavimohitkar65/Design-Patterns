Bridge Pattern Explanation

The Bridge pattern splits a large class or set of related classes into separate hierarchies:

  • Abstraction (Remote Control): Defines the interface and maintains reference to the implementation
  • Implementation (Device): Defines the interface for implementation classes
// Implementation
interface Device {
    boolean isEnabled();
    void enable();
    void disable();
    int getVolume();
    void setVolume(int volume);
}

// Concrete Implementations
class TV implements Device {
    // Implementation specific to TV
}

class Radio implements Device {
    // Implementation specific to Radio
}

// Abstraction
class RemoteControl {
    protected Device device;

    RemoteControl(Device device) {
        this.device = device;
    }

    void togglePower() {
        if (device.isEnabled()) {
            device.disable();
        } else {
            device.enable();
        }
    }

    void volumeUp() {
        device.setVolume(device.getVolume() + 1);
    }
}

// Refined Abstraction
class AdvancedRemoteControl extends RemoteControl {
    void mute() {
        device.setVolume(0);
    }
}

Key Benefits:

  • Separates abstraction from implementation
  • Allows both hierarchies to evolve independently
  • Enables runtime switching of implementations
  • Promotes loose coupling between abstractions and implementations
